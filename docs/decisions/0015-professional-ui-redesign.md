# Decision 0015: Professional UI Redesign

## Status

Accepted

## Date

2026-05-23

## Decision

Redesign Difftray as a restrained professional review console for engineers.

The design brief in `docs/professional-ui-redesign.md` is the normative direction for the next UI implementation pass. It refines and supersedes the visual details in Decision 0006 while keeping the styling stack from that decision:

- CSS Modules
- CSS custom properties
- Radix UI primitives where accessible behavior is useful
- lucide-react icons for standard actions
- no broad component framework

The redesign should use a neutral graphite workspace, subdued operational accent, dense multi-pane layout, real bundled fonts, compact controls, and clear review-state semantics. The interface should be inspired by the Codex app's calm developer-workflow command center, but it should not copy Codex branding or expand Difftray into an agent tool.

## Context

Difftray's current implemented UI can read as too playful and game-like. The strongest causes are dramatic gradient backgrounds, bright amber as a primary identity color, decorative glyphs, large brand treatment, badge-like metadata, and visual emphasis that competes with the diff and review state.

The product needs the opposite signal. Engineers should trust Difftray as a quiet local tool for reviewing changes, especially changes generated by themselves or coding agents. The app should make review state, invalidation, file paths, and diffs easy to scan over long sessions.

Codex app public materials describe a focused workspace for supervising developer workflows, reviewing changes, moving across files and terminals, and keeping work organized by project and thread. Difftray should borrow that level of density and restraint while staying within its own product scope: local diff review only.

## Consequences

Positive:

- Gives future UI work a concrete target instead of a vague "professional" request.
- Makes review state the primary visual signal.
- Reduces risk of arcade, dashboard, or landing-page aesthetics.
- Keeps the implementation compatible with the current stack.
- Preserves Difftray's local review scope.

Negative:

- Requires reworking the current app shell and theme tokens.
- May make the first redesign feel less visually distinctive until the details are tuned.
- Requires visual verification discipline because subtle professional UI depends on spacing, contrast, and alignment.

## Follow-Up Work

- Redesign the app shell around project rail, file queue, diff canvas, and optional inspector.
- Replace the amber-heavy theme with the neutral token set from the design brief.
- Remove decorative gradients, oversized brand treatment, and glyph-like visual elements.
- Rework file and project rows for dense scanning.
- Verify the redesigned app visually at common desktop window sizes.
